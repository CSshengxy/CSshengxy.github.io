从前一直都是用 Visual Studio 这样的集成的比较好的编译器编写一些简单的 C++ 课程作业, 开始接触 C++ 项目发现主要都在用CMake来编写配置文件, 当然也是因为比较喜欢在Ubuntu系统下配置环境编程hhh, 因此准备相对系统的学习一下 CMake 配置文件编写方法。从朋友那里拷贝了一个印度程序员的 CMake 讲解视频, 讲的比较细致。结合 CMake 官方文档在这篇博客中做一个整理记录。

CMake 用来生成编译程序的 Makefile 文件。CMake 具有跨平台的特点, 即同一个CMake配置文件可以针对不同平台生成相应的 Makefile 文件。使用 CMake 生成可执行文件的流程为: ① 创建 CMakeLists.txt 配置文件。② 执行 cmake <PATH> 指令编译生成 Makefile 文件(<PATH> 为 CMakeLists.txt 所在的路径)。③ 执行 make 指令生成可执行文件。

CMake 基本指令

add_executable(EXEC-NAME SOURCE-FILES) 依靠指定的源文件创建可执行文件

project(PROJECT-NAME OTHER-INFO)设置项目的名字和基础信息

cmake_minimum_required(VERSION <min>) 设置项目需要的CMake最小版本

假设一个项目具有如下的文件结构。主函数所在的 main.cpp 文件捕获命令行的两个整数输入, 通过调用 addition.cpp 文件中的 add 函数做加法运算, 调用 division.cpp 文件中的 div 做除法运算, 调用 print_result.cpp 文件中的 print 函数将运算结果输出到控制台。三个 .cpp 源文件的函数均在 .h 头文件中进行了声明, main.cpp 中通过 #include "**.h" 引入三个头文件。

├── build
├── main.cpp

├── addition.cpp
├── division.h
├── division.cpp
├── print_result.h

修改项目文件结构如下。一个”简单”的 CMakeLists.txt 的修改方式为将 add_library 中相应的 addition.cpp 修改为相对路径 my_math_dir/addition.cpp。然而这对于比较大的项目来说反而会使配置文件编写复杂化。因此我们介绍CMake路径相关的指令函数。

project
├── CMakeLists.txt
├── build
├── main.cpp
├── my_math_dir

│   ├── division.h
│   ├── addition.cpp
│   └── division.cpp
└── my_print_dir
    ├── print_result.h
    └── print_result.cpp

add_subdirectory(SOURCE_DIR) 为当前的编译环境添加一个子目录

当源文件和头文件处于同一个文件夹的时候, 编译器可以直接找到相应的头文件, 不需要进行配置, 但当原文件和头文件不在同一个目录的情况下(很多项目都是将头文件放在 include 目录下, 原文件放在 src 目录下), 那么就需要我们在配置文件中明确指出头文件的查找路径。新的文件结构如下。

project
├── CMakeLists.txt
├── build
├── main.cpp
├── my_math_dir
│   ├── include
│   │   ├── addition.h
│   │   └── division.h
│   ├── src
│   │   ├── addition.cpp
│   │   └── division.cpp
│   └── CMakeLists.txt
└── my_print_dir
    ├── include
    │   └── print_result.h
    ├── src
    │   └── print_result.cpp
    └── CMakeLists.txt

target_include_directories(TARGET <scope> DIR1 DIR2 ...) 指明编译目标文件所需的头文件路径
